The unlicensed user report is useful because not every unlicensed identity is a problem, but every unlicensed identity deserves context. Some are expected guests, some are disabled accounts, and some are real provisioning gaps.

## What to review first

Start by filtering for active internal members with no assigned licenses. That group is usually the highest-value follow-up set.

## Why this report is operationally useful

- It helps service teams detect missed onboarding actions.
- It helps security teams spot stale identities.
- It helps licensing teams understand where coverage policies are not being followed consistently.

## Export guidance

CSV works well when another control process or downstream spreadsheet already exists. JSON is better when the result needs to feed custom automation.
